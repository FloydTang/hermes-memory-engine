# Hermes Memory Engine — 记忆条目格式

## 基本格式

```
[标签][优先级] 内容 — 关键细节
```

示例：
```
[行为][P0] 报错不可停——继续试不同思路找workaround
[核心][P0] 服务库: /path/ — 产品线①AI进化之旅②外贸会员...
[平台][P1] trade.tang92.com SPA需飞书OAuth
[群规][P1] AI+外贸年度会员: 飞书群学习, 微信群答疑
```

## 标签一览

| 标签 | 优先级 | 说明 | 保留策略 |
|------|--------|------|----------|
| `[行为]` | P0 | 用户对 agent 行为方式的期望和纠正 | 永不清除 |
| `[核心]` | P0 | 产品线、路径、绝对不能忘的判断 | 永不清除 |
| `[身份]` | P0 | 用户个人资料（名字、偏好、习惯） | 永不清除 |
| `[平台]` | P1 | 外部平台信息、工具配置 | 可归档 |
| `[群规]` | P1 | 群组规则、服务范围 | 可归档 |
| `[infra]` | P1 | 基础设施、文件路径、归档指针 | 可归档 |
| `[架构]` | P1 | 系统架构决策 | 可归档 |
| `[备忘]` | P2 | 临时待办 | 完成后清除 |
| `[待办]` | P2 | 待跟进事项 | 完成后清除 |
| `[temp]` | — | 临时标记，不参与归档 | 使用后立即删除 |

## 优先级规则

- **P0**: 任何时候不可删除或修改。压缩/归档操作跳过 P0 条目
- **P1**: 使用率 >70% 时可归档至 L2 冷存储。同标签条目可合并压缩
- **P2**: 完成后及时清除，最长保留不超过 7 天

## 编写规范

### 好条目 vs 差条目

```
✅ [核心][P0] 产品线: ①AI进化之旅 ②外贸会员(养虾→Codex) ③TikTok陪跑营 ④线下课
❌ [核心][P0] 我们目前有四个产品线，第一个是一人企业的AI进化之旅，它主要面向那些...

✅ [行为][P0] 报错不可停——继续试不同思路。案例: docs +update --mode append 绕过wiki写权限
❌ [行为][P0] 如果遇到错误的时候，不要停下来，要继续尝试其他的方法，比如之前有一次...

✅ [平台][P1] trade.tang92.com SPA需飞书OAuth。提取: browser_console("body.innerText")
❌ [平台][P1] tang92这个平台是一个单页应用，需要飞书授权才能登录，当我们想获取内容的时候...
```

### 原则

1. **判断先行** — 第一句直接给结论
2. **详情用破折号** — `— 关键细节`，不要写长句
3. **保留核心参数** — URL、路径、命令、API 名，一个字不能省
4. **去掉废话** — 删掉"我认为""我觉得""可能""比较"类修饰词
5. **优先用名词** — 用"产品线: A/B/C" 替代 "我们有三个产品线，分别是..."
